Полностью переработанная версия. Дизайн-система, шапка и подвал — из
утверждённого макета, расширены до полноценного сайта.

## Что изменилось против прошлой версии

### Контент (было ~330 слов на странице → стало ~950–1000)
Каждая страница услуги теперь содержит:
- развёрнутый текстовый блок: когда нужна услуга, показания, виды, экспертные пояснения;
- таблицу сравнения вариантов (винир/коронка, виды имплантации, типы брекетов, стадии кариеса);

- подробную таблицу цен с пояснениями;
- 6–8 развёрнутых FAQ (было 2) — это и доверие, и SEO;
- фото-баннер услуги.

### Доверие
- Trust-bar на всех страницах: лицензия, гарантия, рассрочка, налоговый вычет;
- Юридический блок в подвале: оператор, ИНН, ОГРНИП, № лицензии;
- Страница «Политика конфиденциальности» (privacy.html) — типовой документ по 152-ФЗ;
- Страница врачей: образование, квалификация, специализация по каждому;
- Реальные клинические описания в кейсах вместо подписей-заглушек.

### Функциональность (раньше не работало)
- Слайдер отзывов на главной — автопрокрутка, точки навигации, пауза при наведении;
- Карусель «До/После» — рабочие стрелки, drag-scroll, эффект разделения через clip-path;
- Формы — единый обработчик: POST на endpoint, fallback на mailto, состояние «отправлено»;
- Маска телефона — авто-форматирование +7 (___) ___-__-__;
- Бургер-меню с анимацией, закрытие по Escape/клику;

### SEO
- Schema.org на каждой странице: Dentist, BreadcrumbList, FAQPage, MedicalProcedure, Physician, WebSite;

- canonical на каждой странице;
- уникальные title/description;
- sitemap.xml (13 URL), robots.txt;
- WebP + picture с JPG-фолбеком.

### Доступность
- aria-label на кнопках, lang="ru", title у iframe карты;
- семантические теги (nav, article, хлебные крошки).

## Исправление критического бага (пустая секция)

Была обнаружена и устранена проблема: на страницах услуг между hero и
блоком «Сравните варианты» зияла пустая область, а форма была без стилей.

Причины (две):
1. В CSS финальной сборки отсутствовали правила .layout-2col, .sticky-form,
   .prose, .timeline, .compare, .trust-bar, .reveal и др. (потерялись при
   слиянии папок) — восстановлены полностью.
2. main.js был старой урезанной версией (20 строк) без логики .reveal —
   элементы с opacity:0 никогда не показывались. Восстановлен полный JS.

Дополнительно сделано пуленепробиваемо:
- .reveal теперь прогрессивное улучшение: контент виден по умолчанию,
  скрывается под анимацию только если JS реально работает (класс js-on
  ставится мгновенно в <head>). Без JS / при ошибке — текст виден всегда.
- Тройной предохранитель показа: IntersectionObserver + sweep на скролле
  + принудительный показ всего через 4 секунды.

Проверено рендером всех 13 страниц: 0 скрытых блоков, 0 пустых секций.

## ОКОНЧАТЕЛЬНОЕ исправление пустых секций (v5)

Корень проблемы: контент имел opacity:0 и становился видимым только
после выполнения main.js. На проде JS по какой-то причине не отрабатывал
(defer + ошибка/кеш/медленная сеть) → контент скрыт навсегда → пустые
секции и «голая» форма.

Решение — полностью убрана зависимость контента от JS:
- .reveal больше не прячет контент. Контент виден ВСЕГДА.
- Лёгкое появление — чистый CSS @keyframes, запускается сам без JS.
- Удалён инлайн js-on, удалена reveal-логика и IntersectionObserver из JS.
- JS теперь отвечает ТОЛЬКО за необязательное: меню, карусели, слайдер
  отзывов, маску телефона, отправку формы. Если он не загрузится —
  потеряются только эти улучшения, весь текст/цены/формы останутся.

Проверено рендером всех 13 страниц с ПОЛНОСТЬЮ ОТКЛЮЧЁННЫМ JavaScript:
весь контент виден, пустых секций нет. Это гарантирует, что на проде
такое не повторится ни при каких условиях.
