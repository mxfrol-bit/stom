# Стоматология Натальи Боровиковой — сайт

Статический сайт (чистый HTML/CSS/JS, без сборки). Готов к публикации на **GitHub Pages** — достаточно загрузить файлы в репозиторий.

## Как опубликовать на GitHub Pages

1. Создайте репозиторий и загрузите в него **всё содержимое этой папки** (включая скрытый файл `.nojekyll`).
2. В репозитории: **Settings → Pages → Build and deployment → Source: Deploy from a branch**, ветка `main`, папка `/ (root)`.
3. Через 1–2 минуты сайт откроется по адресу `https://<логин>.github.io/<репозиторий>/`.
4. (Опционально) свой домен — см. ниже.

> Все пути в проекте **относительные**, поэтому сайт одинаково работает и на `github.io`, и на своём домене, и в подпапке.

### Свой домен (bor-dent.ru)
Создайте в корне файл `CNAME` с одной строкой:
```
bor-dent.ru
```
и пропишите DNS по инструкции GitHub Pages.

## Важно про надёжность
Шапка и подвал **вшиты в каждую страницу** (не подгружаются через JS). Поэтому контент, меню и форма видны всегда — даже если JavaScript не загрузится. JS отвечает только за улучшения: слайдеры, модалки врачей, маску телефона, отправку формы, анимации.

```

├── gigiena.html             Услуга: Гигиена и чистка
├── brekety.html             Услуга: Брекеты и элайнеры
├── detskaya-stomatologiya.html  Услуга: Детская стоматология
├── vrachi.html              Врачи (модалки с биографией)
├── ceny.html                Цены
├── otzyvy.html              Отзывы + «До/После»
├── kontakty.html            Контакты, карта, форма записи
├── privacy.html             Политика конфиденциальности (152-ФЗ)

    ├── js/main.js
```

Каждая страница услуги — самостоятельная и полностью расписана: hero с ценой, преимущества, этапы лечения, технологии, прайс, FAQ и CTA. Они связаны из карточек на главной, из выпадающего меню «Услуги» в шапке, из футера и из `sitemap.xml`.

## Фишки и анимации
- **Прелоудер** — экран загрузки с анимированным логотипом. Безопасен: уходит сам через CSS-анимацию, даже если JS не сработает (контент не блокируется).
- **Вау-баннер** — градиентный текст, аврора-свечение, лёгкий параллакс карточек на движение мыши, бейдж рейтинга. Всё отключается при `prefers-reduced-motion`.
- **Инфографика** — блок «Этапы» и тёмная полоса с анимированными счётчиками достижений.
- **Иконки** — единый линейный SVG-спрайт (`assets/img/icons.svg`).

## SEO по Ворсме
- Geo-метатеги (Ворсма, Нижегородская область, координаты) на всех страницах.
- Микроразметка JSON-LD: `Dentist` с зоной обслуживания (Ворсма, Павлово, Богородск, Сосновское), а на каждой услуге — `MedicalProcedure` + `BreadcrumbList` + `FAQPage`.
- `FAQPage` на главной + локальный SEO-блок с зоной обслуживания.

## Заменить перед публикацией (плейсхолдеры)

| Где | Что заменить |
|---|---|
| Подвал, privacy.html | ИНН `520200000000`, ОГРНИП `320527500000000`, лицензия `ЛО-52-01-007890`, юр. наименование |
| Trust-bar (в шапке) | № лицензии, при необходимости — текст пунктов |
| Подвал, контакты | реальные ссылки соцсетей VK / WhatsApp / Telegram (сейчас `#`) |
| `kontakty.html` | координаты карты (сейчас Ворсма, ул. Ленина, 35) |
| `assets/js/main.js` | endpoint формы — сейчас Supabase; при необходимости заменить |
| Тексты, цены, фото врачей, отзывы | на реальные данные |

`privacy.html` — типовой документ-образец. Перед публикацией проверьте у юриста и подставьте реальные реквизиты: сбор телефонов без действующей политики нарушает 152-ФЗ.

> Файлы `package.json` / `railway.json` нужны только для альтернативного хостинга (Railway) и на GitHub Pages не используются — можно удалить.
